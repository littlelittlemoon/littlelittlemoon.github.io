CV | Algorithm | Harris Corner Detector Principle and Implementation

Kayleen · 2020-5-29 · 次阅读

前言

一般局部特征匹配涉及以下三个步骤：

  1. 检测：找到一组独特的兴趣点（ 🐻 本文算法解决的问题）；
  2. 描述：提取每个兴趣点周围的特征描述符作为矢量；
  3. 匹配：计算特征向量之间的距离以找到对应关系。

这篇文章主要介绍检测兴趣点的一个简单算法: Harris Corner Detector。主要内容包括该算法的原理介绍和python的实现方法。

Harris Corner Detector Principle and Implementation

Harris Corner Detector

拐角 (Corner)

Harris拐角检测算法（也称为Harris＆Stephens拐角检测器）是可用的最简单的拐角检测器之一。拐角是一个点，其局部邻域位于两个主要且不同的边缘方向。换句话说，一个角可以解释为两个边缘的交点，其中边缘是图像亮度的突然变化。角是图像中的重要特征，通常被称为兴趣点，它们对平移，旋转和照明不变。

因此，让我们理解为什么拐角被认为是更好的特征或对特征块匹配有用。在上图中，如果我们选取平坦区域，则在任何方向都不会观察到梯度变化。类似地，在边缘区域中，沿边缘方向没有观察到梯度变化。因此，平坦区域和边缘区域都不利于色块匹配，因为它们的区别不大（沿边缘区域中的边缘存在许多相似的色块）。在拐角区域，我们观察到在所有方向上的明显梯度变化。由于这个角被认为对特征块匹配有益（在任何方向上移动窗口都会产生很大的外观变化），并且通常在视点变化时更稳定。

拐角检测 (Corner Detection)

从上述拐角的概念，我们很容易可以想到的是考虑图像中每个像素p周围的小窗口。要确定所有此类唯一的像素窗口，可以通过在给定方向上将每个窗口少量移动并测量像素值中发生的变化量来判断。故算法的基本思想是找到(u, v)在所有方向上的位移的强度差异，表示如下：

E(u,v)=∑x,yw(x,y)⏟window function[I(x+u,y+v)⏟shifted intensity−I(x,y)⏟intensity]2 E(u, v) = \sum_{x,y} \underbrace{w(x, y)}_{\text{window function}} [ \underbrace{I(x+u, y+v)}_{\text{shifted intensity}} - \underbrace{I(x, y)}_{\text{intensity}}]^2 E(u,v)=x,y∑​window functionw(x,y)​​[shifted intensityI(x+u,y+v)​​−intensityI(x,y)​​]2

E(u,v)≈[uv]M[uv] E(u, v) \approx \begin{bmatrix} u & v \end{bmatrix} M \begin{bmatrix} u \\ v \end{bmatrix} E(u,v)≈[u​v​]M[uv​]

M=∑x,yw(x,y)[IxIxIxIyIxIyIyIy] M = \sum_{x,y}w(x,y) \begin{bmatrix} I_xI_x & I_xI_y \\ I_xI_y & I_yI_y \end{bmatrix} M=x,y∑​w(x,y)[Ix​Ix​Ix​Iy​​Ix​Iy​Iy​Iy​​]

IxI_xIx​和IyI_yIy​分别是xxx和yyy方向的图像导数。可以使用sobel内核来计算导数。

请记住，我们希望差得平方和（第二项）在所有八个方向上的偏移都较大。或者所有方向而言，第二项都应较小。通过求解M的特征向量，我们可以获得第二项最大和最小增加的方向。相应的特征值表示增加的实际值。为每个窗口计算得分R：

R=det⁡M−k(trace M)2det⁡M=λ1λ2trace M=λ1+λ2 R = \det M - k(trace \space M)^2 \\ \det M = \lambda_1 \lambda_2 \\ trace \space M = \lambda_1 + \lambda_2 R=detM−k(trace M)2detM=λ1​λ2​trace M=λ1​+λ2​

其中λ1\lambda_1λ1​和λ2\lambda_2λ2​为MMM的特征值。因此，这些特征值的值决定了区域是角，边还是平面：

  • 当∣R∣|R|∣R∣较小，即λ1\lambda_1λ1​和λ2\lambda_2λ2​较小，该区域平坦；
  • 当R<0R<0R<0时，即λ1>>λ2\lambda_1 >> \lambda_2λ1​>>λ2​或λ1<<λ2\lambda_1 << \lambda_2λ1​<<λ2​，该区域为边；
  • 当R大时，即λ1\lambda_1λ1​和λ2\lambda_2λ2​大，且λ1 λ2\lambda_1 ~ \lambda_2λ1​ λ2​时，该区域是角。

算法的过程

  1. 将彩色图转为灰度图；
  2. 空间导数计算；
  3. 结构张量设置；
  4. 计算Harris分数；
  5. 检测拐角；

Implementation

将彩色图转为灰度图

from skimage import io, img_as_float32
from skimage.color import rgb2gray

image_color = img_as_float32(io.imread('image_file_path'))
image = rgb2gray(image_color)

空间导数计算

import cv2

# 计算Ix, Iy
Ix = cv2.Sobel(image, cv2.CV_64F, 1, 0, ksize=5)
Iy = cv2.Sobel(image, cv2.CV_64F, 0, 1, ksize=5)

结构张量设置

from scipy.ndimage import gaussian_filter

# 计算(Ix)^2, (Iy)^2, Ixy
Ixx = gaussian_filter(Ix ** 2, sigma=1)
Ixy = gaussian_filter(Iy * Ix, sigma=1)
Iyy = gaussian_filter(Iy ** 2, sigma=1)

计算Harris分数

这里有用到变量alpha，alpha是将角点与边缘分开的灵敏度因子，通常为接近零的值，这里我把alpha设置为0.04。alpha的值较小会更容易检测到尖角。harris_score是图像中每行峰值的数组。可以使用这些峰值来区分拐角和边。

# 计算行列式 A 和 trace A
detA = Ixx * Iyy - Ixy ** 2
traceA = Ixx + Iyy

alpha = 0.04

# 计算Harris's score
harris_score = detA - alpha * traceA ** 2

检测拐角

在检测拐角的时候我是直接用的peak_local_max函数。我在这里设置了一个参数：threshold_rel，这个值是用来检测峰时，峰的最小强度（max(image)∗thresholdrelmax(image)*threshold_relmax(image)∗thresholdr​el）。更改这个参数会影响拐角检测效果。具体可以参考文档。

使用 peak_local_max 函数

from skimage.feature import peak_local_max

# find corners(这里请忽视注释)
# 0.06 => 65%(notre_dame)|23%(mt_rushmore)|0%(e_gaudi)
# 0.05 =>notre_dame： 73%(0.8)/72%(0.7)|mt_rushmore：26%((ratio：0.8))/66%(ratio：0.7)|0%(e_gaudi)
# 0.04 =>notre:75%(0.8)/83%(0.7)/75%(0.6)/83%(0.6,min_dis=5)/66%(0.7,min5)61%(0.8)|50%（0.7）/18%(0.8)|
# 0.03 =>67%(0.8)/78%(0.7)
corners = peak_local_max(harris_score, threshold_rel=0.04)

通过R判断

这里也可以通过上述原理中的R来判断，但是效果不是很好，应该是阈值设置和峰值选取距离设置的问题（我这里没有设置峰值距离）。具体实现如下：

thresh = 0.04 * harris_score.max()

for r_i, score in enumerate(harris_score):
    for c_i, s in enumerate(score):
        if s > thresh:
            # corner
            corners.append([r_i, c_i])

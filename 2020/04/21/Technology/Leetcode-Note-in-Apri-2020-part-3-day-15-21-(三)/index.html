Minimum Path Sum

Given a m x n grid filled with non-negative numbers, find a path from top left to bottom right which minimizes the sum of all numbers along its path.
Note: You can only move either down or right at any point in time.
Example:
Input:
[
[1,3,1],
[1,5,1],
[4,2,1]
]
Output: 7
Explanation: Because the path 1→3→1→1→1 minimizes the sum.

题目大意是找到从左上角到右下角的最短路径长度。这道题很明显是个动态规划问题，但是我一开始并没有想到这个问题，还是练太少了。首先我们先回顾一下动态规划算法。

动态规划

动态规划（英语：Dynamic programming，简称DP）是一种在数学、管理科学、计算机科学、经济学和生物信息学中使用的，通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。
动态规划常常适用于有重叠子问题和最优子结构性质的问题，动态规划方法所耗时间往往远少于朴素解法。
动态规划背后的基本思想非常简单。大致上，若要解一个给定问题，我们需要解其不同部分（即子问题），再根据子问题的解以得出原问题的解。
通常许多子问题非常相似，为此动态规划法试图仅仅解决每个子问题一次，从而减少计算量：一旦某个给定子问题的解已经算出，则将其记忆化存储，以便下次需要同一个子问题解之时直接查表。这种做法在重复子问题的数目关于输入的规模呈指数增长时特别有用。

适用情况

  1. 最优子结构性质。如果问题的最优解所包含的子问题的解也是最优的，我们就称该问题具有最优子结构性质（即满足最优化原理）。最优子结构性质为动态规划算法解决问题提供了重要线索。
  2. 无后效性。即子问题的解一旦确定，就不再改变，不受在这之后、包含它的更大的问题的求解决策影响。
  3. 子问题重叠性质。子问题重叠性质是指在用递归算法自顶向下对问题进行求解时，每次产生的子问题并不总是新问题，有些子问题会被重复计算多次。动态规划算法正是利用了这种子问题的重叠性质，对每一个子问题只计算一次，然后将其计算结果保存在一个表格中，当再次需要计算已经计算过的子问题时，只是在表格中简单地查看一下结果，从而获得较高的效率。

实例

背包问题

背包问题（Knapsack problem）是一种组合优化的NP完全问题。问题可以描述为：给定一组物品，每种物品都有自己的重量和价格，在限定的总重量内，我们如何选择，才能使得物品的总价格最高。问题的名称来源于如何选择最合适的物品放置于给定背包中。

Solution

这道题的解题思路类似于背包问题，只不过我们是找最短路径。

假设用一个path变量记录当前的最短路径，i, j为指向当前遍历点的index
我们可以通过以下四种情况来讨论：

  1. i=0, j=0: 是起点
  2. i=0, j>0: 由于我们只能往下和往右移动，那么上一步只能从左边来，所以经过该点的最短路径只能是: path += grid[i][j-1];
  3. i>0, j=0: 同理，上一步只能从上边来，所以经过改点的最短路径是: path += grid[i-1][j];
  4. i>0, j>0: 上一步既可以从左边来，也可以从上边来，这时需要判断从哪边过来加上当前的值会是最小，就能得到经过当前点的最短路径是多少，故可表示为: path = min(grid[i][j-1], grid[i-1][j]).

等到遍历到右下角的那个点，我们就可以得到从左上角到右下角的最短路径了。这里就是用动态规划的思路，讲一个复杂的大问题（从左上角到右下角的最短路径）划分为小的问题（到每一个点的最短路径），然后通过不断迭代解决小问题得到大问题的解（所有最短路径加起来就是最终要求的最短路径）。具体实现请看代码。

class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        len_0 = len(grid)
        len_1 = len(grid[0])
        for i in range(len_0):
            for j in range(len_1):
                if i == 0 and j == 0:
                    continue
                elif i == 0:
                    grid[i][j] += grid[i][j-1]
                elif j == 0:
                    grid[i][j] += grid[i-1][j]
                else:
                    grid[i][j] = min(grid[i][j-1], grid[i-1][j]) + grid[i][j] 
                
        return grid[-1][-1]

Search in Rotated Sorted Array

Suppose an array sorted in ascending order is rotated at some pivot unknown to you beforehand.
(i.e., [0,1,2,4,5,6,7] might become [4,5,6,7,0,1,2]).
You are given a target value to search. If found in the array return its index, otherwise return -1.
You may assume no duplicate exists in the array.
Your algorithm’s runtime complexity must be in the order of O(log n).
Example 1:
Input: nums = [4,5,6,7,0,1,2], target = 0
Output: 4
Example 2:
Input: nums = [4,5,6,7,0,1,2], target = 3
Output: -1

Solution

这道题有一个限制，就是时间复杂度必须为O(logn)，那我的第一想法还是用二分法。虽然原数组并不是有序的，但是存在局部有序，我们可以用二分法来探测哪一部分有序，如果所要找的taget在有序的部分，那么我们可以直接在该部分使用二分法查找。如果不在，那么继续重复上述步骤，直到找到target所在的有序的部分。如果找不到则返回-1。

class Solution:
    def search(self, nums: List[int], target: int) -> int:
        size = len(nums)
        if size == 0:
            return -1

        left = 0
        right = size - 1
        while left < right:
	        # 取左中位数
            mid = left + (right-left) // 2
            # 如果左半部分有序
            if nums[left] < nums[mid]:
                # 如果target可能在该段有序数字内，则直接在该段有序序列中使用二分查找
                if nums[left] <= target <= nums[mid]:
                    right = mid
                # 如果target不在左边有序序列，则可能在右半部分（那么left=mid+1）
                # 在右半部分继续找target可能存在的有序序列
                else:
                    left = mid + 1
            # 如果左半部分无序，那么右半部分一定有序
            else:
		        # 为了使这里left和right的更新和上面一样，所以使用nums[mid+1]
                if nums[mid+1] <= target <= nums[right]:
                    left = mid + 1 
                # 如果target不在右边有序序列，则可能在左半部分（那么right=mid）
                # 在左半部分继续找target可能存在的有序序列
                else:
                    right = mid
        
        return left if nums[left] == target else -1

Construct Binary Search Tree from Preorder Traversal

Return the root node of a binary search tree that matches the given preorder traversal.
(Recall that a binary search tree is a binary tree where for every node, any descendant of node.left has a value < node.val, and any descendant of node.right has a value > node.val. Also recall that a preorder traversal displays the value of the node first, then traverses node.left, then traverses node.right.)
Example 1:
Input: [8,5,1,7,10,12]
Output: [8,5,10,1,7,null,12]

Solution

题目是要求是，我们根据给定的数据，构造二叉搜索树。可以采用递归的方法进行二叉搜索树的构建。

这里的核心思路是：指定一个bound变量，记录当前需要处理的最大值，初始化为inf，使：

  1. 左递归的所有元素都小于当前node的value；
  2. 右递归的所有元素都小于当前这个bound值。
# Definition for a binary tree node.
class TreeNode:
    def __init__(self, x):
        self.val = x
        self.left = None
        self.right = None

class Solution:
    i = 0
    
    # 指定一个最大数据：bound=float('inf')
    def bstFromPreorder(self, A, bound=float('inf')):
        
        # 左递归将采用小于root.val的元素
        # 右递归将采用其余小于bound的元素
        if self.i == len(A) or A[self.i] > bound:
            return None
        
        root = TreeNode(A[self.i])
        self.i += 1
        root.left = self.bstFromPreorder(A, root.val)
        root.right = self.bstFromPreorder(A, bound)
        return root

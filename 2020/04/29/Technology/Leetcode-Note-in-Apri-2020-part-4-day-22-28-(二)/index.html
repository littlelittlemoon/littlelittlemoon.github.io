First Unique Number

You have a queue of integers, you need to retrieve the first unique integer in the queue.
Implement the FirstUnique class:
- FirstUnique(int[] nums) Initializes the object with the numbers in the queue.
- int showFirstUnique() returns the value of the first unique integer of the queue, and returns -1 if there is no such integer.
- void add(int value) insert value to the queue.
Example 1:
Input: ["FirstUnique","showFirstUnique","add","showFirstUnique","add","showFirstUnique","add","showFirstUnique"]
[[[2,3,5]],[],[5],[],[2],[],[3],[]]
Output:
[null,2,null,2,null,3,null,-1]
Explanation:
FirstUnique firstUnique = new FirstUnique([2,3,5]);
firstUnique.showFirstUnique(); // return 2
firstUnique.add(5); // the queue is now [2,3,5,5]
firstUnique.showFirstUnique(); // return 2
firstUnique.add(2); // the queue is now [2,3,5,5,2]
firstUnique.showFirstUnique(); // return 3
firstUnique.add(3); // the queue is now [2,3,5,5,2,3]
firstUnique.showFirstUnique(); // return -1

题目大意是实现一个FirstUnique类，包含两个方法：

  1. add: 往数组里面增加元素；
  2. showFirstUnique: 返回当前数组里面第一个唯一值。

Solution

这个题应该算是比较简单的，解题思路也很容易想到，主要就是看怎么优化，使时间复杂度和空间复杂度尽可能的小。我最开始想到的思路是用一个dictionary来存放唯一数，然后希望能排序，所以就用的OrderedDict，但是后面昨晚看讨论的时候，有的小伙伴说python3.6以上已经支持dictionary排序了，所以就没必要用OrderedDict了，然后我就直接用的dictionary。

对于所有元素我最开始还是想着用一个dictionary来存放，后面看了大家的讨论，其实没必要用dictionary存放，可以用一个set存放所有出现过的元素。

因为这个类只有增加元素的方法，没有删减元素的方法，所以只要出现过，就一直存在。这样问题就变得简单了，我们只需要记录每一个唯一元素和以前出现过的元素，如果新增加的元素是以前的唯一元素，那么就从唯一元素的dictionary中删除这个元素。具体看代码实现。

class FirstUnique:
    def __init__(self, nums: List[int]):
        self.uniq = {}
        self.num_set = set()
        for num in nums:
            self.add(num)

    def showFirstUnique(self) -> int:
        for num in self.uniq.keys():
            return num
        
        return -1 

    def add(self, value: int) -> None:
        if value in self.num_set:
            self.uniq.pop(value, 1)
        else:
            self.num_set.add(value)
            self.uniq[value] = 1

Longest Common Subsequence

Given two strings text1 and text2, return the length of their longest common subsequence.
A subsequence of a string is a new string generated from the original string with some characters(can be none) deleted without changing the relative order of the remaining characters. (eg, "ace" is a subsequence of "abcde" while "aec" is not). A common subsequence of two strings is a subsequence that is common to both strings.
If there is no common subsequence, return 0.
Example 1:
Input: text1 = "abcde", text2 = "ace"
Output: 3
Explanation: The longest common subsequence is "ace" and its length is 3.
Example 2:
Input: text1 = "abc", text2 = "abc"
Output: 3
Explanation: The longest common subsequence is "abc" and its length is 3.
Example 3:
Input: text1 = "abc", text2 = "def"
Output: 0
Explanation: There is no such common subsequence, so the result is 0.

Solution

题意不难理解，就是找两个字符串中的最大子串长度，但是这个题我做了很久（还需继续努力呀😆），最后看了大家的讨论，然后慢慢找到了比较好的解决方案。

关于这个问题，大家可以看看维基百科

最长公共子序列（LCS）问题是在一组序列（通常只有两个序列）中找到所有序列共有的最长子序列的问题。它不同于最长的常见子字符串问题：与子字符串不同，子序列不需要占据原始序列中的连续位置。最长的常见子序列问题是经典计算机科学问题，它是数据比较程序（例如diff实用程序）的基础，并已在计算语言学和生物信息学中得到应用。它也被修订控制系统（例如Git）广泛使用，用于协调对修订控制的文件集合所做的多项更改。例如，考虑序列（ABCD）和（ACBAD）。它们具有5个长度为2的公共子序列：（AB），（AC），（AD），（BD）和（CD）；2个长度为3的常见子序列：（ABD）和（ACD）；不再是常见的子序列。因此（ABD）和（ACD）是它们最长的公共子序列。

解题思路（动态规划）

LCS问题具有最佳子结构：可以将问题分解为更小，更简单的子问题，再将问题分解为更简单的子问题，依此类推，直到最终解决方案变得无关紧要。
LCS具有重叠的子问题：高级别子问题的解决方案经常重用低级别子问题的解决方案。具有这两个属性的问题适合于动态编程方法，在这种方法中，子问题的解决方案得以记忆，即子问题的解决方案被保存以供重用。

Traceback approach

计算LCS表的一行的LCS仅需要当前行和上一行的解决方案。但是，对于长序列，这些序列可能会变得很长而且很长，需要大量的存储空间。如下表所示，不保存实际的子序列，而是保存子序列的长度和箭头的方向，可以节省存储空间。

从表中的最后一个单元格开始，通过向后跟随箭头的“回溯”过程推导出实际的子序列。当长度减小时，序列必须具有一个公共元素。当单元格中显示两个箭头时，可能有几种路径。下表是进行此类分析的表格，其中长度将要减少的单元格中带有数字。粗体数字表示序列(GA)(GA)(GA)。

伪代码

计算LCS的长度
function LCSLength(X[1..m], Y[1..n])
    C = array(0..m, 0..n)
    for i := 0..m
        C[i, 0] = 0
    for j := 0..n
        C[0, j] = 0
    for i := 1..m
        for j := 1..n
            if X[i] = Y[j] //i-1 and j-1 if reading X & Y from zero
                C[i, j] := C[i-1, j-1] + 1
            else
                C[i, j] := max(C[i, j-1], C[i-1, j])
    return C[m, n]
读出LCS
function backtrack(C[0..m,0..n], X[1..m], Y[1..n], i, j)
    if i = 0 or j = 0
        return ""
    if  X[i] = Y[j]
        return backtrack(C, X, Y, i-1, j-1) + X[i]
    if C[i, j-1] > C[i-1, j]
        return backtrack(C, X, Y, i, j-1)
    return backtrack(C, X, Y, i-1, j)
读出所有LCS
function backtrackAll(C[0..m,0..n], X[1..m], Y[1..n], i, j)
    if i = 0 or j = 0
        return {""}
    if X[i] = Y[j]
        return {Z + X[i] for all Z in backtrackAll(C, X, Y, i-1, j-1)}
    R := {}
    if C[i,j-1] ≥ C[i-1,j]
        R := R ∪ backtrackAll(C, X, Y, i, j-1)
    if C[i-1,j] ≥ C[i,j-1]
        R := R ∪ backtrackAll(C, X, Y, i-1, j)
    return R

减少存储空间

算法中的C矩阵随着序列的长度平方增长。对于两个100个项目的序列，将需要10,000个项目的矩阵，并且需要进行10,000个比较。在大多数实际情况下，尤其是源代码差异和补丁，文件的开头和结尾很少更改，并且几乎可以肯定不会同时更改。如果序列中间仅更改了几项，则可以消除开始和结束。这不仅减少了矩阵的存储要求，而且减少了必须进行的比较次数。

function LCS(X[1..m], Y[1..n])
    start := 1
    m_end := m
    n_end := n
    trim off the matching items at the beginning
    while start ≤ m_end and start ≤ n_end and X[start] = Y[start]
        start := start + 1
    trim off the matching items at the end
    while start ≤ m_end and start ≤ n_end and X[m_end] = Y[n_end]
        m_end := m_end - 1
        n_end := n_end - 1
    C = array(start-1..m_end, start-1..n_end)
    only loop over the items that have changed
    for i := start..m_end
        for j := start..n_end
            the algorithm continues as before ...

在最佳情况下，即序列无变化，这种优化将完全消除对C矩阵的需求。在最坏的情况下，如果更改序列中的第一个和最后一个项目，则仅执行两个附加比较。

代码实现

class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m, n = len(text1), len(text2)
        if m < n:
            return self.longestCommonSubsequence(text2, text1)

        dp = [[0] * (n + 1) for _ in range(2)]
        
        for i, c in enumerate(text1):
            for j, d in enumerate(text2):
                if c == d:
                    dp[1-i%2][j+1] = 1 + dp[i%2][j]
                else:
                    dp[1-i%2][j+1] = max(dp[i%2][j+1], dp[1-i%2][j])
        return dp[m%2][-1]

时间复杂度： O(m * n)
空间复杂度： O(min(m, n))

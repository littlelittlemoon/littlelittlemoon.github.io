Check If a String Is a Valid Sequence from Root to Leaves Path in a Binary Tree

Given a binary tree where each path going from the root to any leaf form a valid sequence, check if a given string is a valid sequence in such binary tree.
We get the given string from the concatenation of an array of integers arr and the concatenation of all values of the nodes along a path results in a sequence in the given binary tree.
Example 1:

Input: root = [0,1,0,0,1,0,null,null,1,0,0], arr = [0,1,0,1]
Output: true
Explanation: The path 0 -> 1 -> 0 -> 1 is a valid sequence (green color in the figure).
Other valid sequences are:
0 -> 1 -> 1 -> 0
0 -> 0 -> 0 even a sequence.
Example 2:

Input: root = [0,1,0,0,1,0,null,null,1,0,0], arr = [0,0,1]
Output: false
Explanation: The path 0 -> 0 -> 1 does not exist, therefore it is not even a sequence.
Example 3:

Input: root = [0,1,0,0,1,0,null,null,1,0,0], arr = [0,1,1]
Output: false
Explanation: The path 0 -> 1 -> 1 is a sequence, but it is not a valid sequence.

Solution

这个题应该算是常规题，直接用深度优先遍历算法遍历树，注意一下递归停止条件就可以了，具体看代码注释吧。

# Definition for a binary tree node.
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right
        
class Solution:
    def isValidSequence(self, root: TreeNode, arr: List[int]) -> bool:
        n = len(arr) - 1
        
        def isValid(root, arr, i):
            # 若当前到达叶结点并且遍历到数组的最后一个元素，则比较最后一个元素是否相等
            if root and i == n and not(root.left) and not(root.right):
                return root.val == arr[i]

            # 若数组未遍历到最后一个元素且树遍历完所有结点，
            # 或者数组遍历完所有元素，但树还没到叶子节点，则无效
            if (i <= n and not(root)) or (i > n and root):
                return False
            
            # 当前节点对应的数字不等于数组相应位置的数字，无效
            if root.val != arr[i]:
                return False

            # 如果数组未遍历完所有元素，则继续比较
            
            left = isValid(root.left, arr, i+1) if i < n else False
            
            # 如果左子树满足条件，则返回结果，否则遍历右子树是否满足条件
            if left:
                return True
            right = isValid(root.right, arr, i+1) if i < n else False
            
            return right
        
        return isValid(root, arr, 0)

Binary Tree Maximum Path Sum

Given a non-empty binary tree, find the maximum path sum.
For this problem, a path is defined as any sequence of nodes from some starting node to any node in the tree along the parent-child connections. The path must contain at least one node and does not need to go through the root.
Example 1:
Input: [1,2,3]

   1  
  / \  
 2   3

Output: 6

Example 2:

Input: [-10,9,20,null,null,15,7]

  -10
  /  \
 9   20
    /  \
   15   7

Output: 42

Solution

对，就是它，不知道来看到的小伙伴能不能看懂这道题的意思，如果一次就看懂了，我真的要给你点个赞👍，反正我是没看懂。。最后去讨论区求助，才慢慢搞懂是要求个啥。

题目大意是：求最大路径和，但是路径的起点和终点可以是任意节点，但不可以重复走某一个节点，我也不知道我说清楚没🤣。

例如：第二个例子最大路径是：15 + 7 + 20 = 42,
因为其他路径都比它小：
- 9 - 10 + 20 + 15 = 34
- 9 - 10 + 20 + 7 = 26
- - 10 + 20 + 15 = 25
- - 10 + 20 + 7 = 17
- ...

现在应该能明白了，这题搞明白题意之后，可以想到用深度优先遍历算法来实现最大路径和的累加，最后确实也是用这个法子。

核心思想是：通过累加局部最大路径和来求解全局最大路径和，具体看代码注释。

# Definition for a binary tree node.
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right
        
class Solution:
    def maxPathSum(self, root: TreeNode) -> int:
        # 递归问题：
        # 全局最大路径和 = 各局部最大路径和（左子树最大路径+根节点权重+右子树最大路径）
        self.max_path = float("-inf")
        
        # 递归计算左右子树中的最大路径，因为路径只能包含一条分支，并与其他分支通过根节点连接；
        # 所以这里返回的值我们可以直接确定为：max(0, max(left,  right) + root.val)；
        # 这里如果计算出负值的可直接舍去，因为它只会让之前累加的最大路径变小；
        # 递归的第一个返回条件是当前节点为None，None节点可贡献的路径为0，然后开始回溯。
        def count_path(root):
            
            # 如果当前node为null，则可回溯的最大路径为0
            if not(root): return 0
            
            # 还是深度优先遍历算法的思想，但是不同的是，根据题目定义核要求计算全局最大路径和，
            # 可通过计算左右最大路径（局部最大），并通过根节点连接成全局最大路径和
            
            # 局部最大：左子树最大路径
            left = count_path(root.left)
            
            # 局部最大：右子树最大路径
            right = count_path(root.right)
            
            # 实时更新可能产生的最大路径和：左子树最大路径+右子树最大路径+根节点权值
            # 与之前的最大路径比较，如果大于之前的最大路径，则将当前最大路径作为最大路径
            self.max_path = max(self.max_path, left + right + root.val)
            
            # 局部最大路径和（路径和：只走一条分支->选最大分支）：回溯当前遍历到的最大路径，
            # 如果出现负值只会让全局路径和更小，故不计（舍->0）
            return max(0, max(left,  right) + root.val)
        
        
        count_path(root)
        return self.max_path

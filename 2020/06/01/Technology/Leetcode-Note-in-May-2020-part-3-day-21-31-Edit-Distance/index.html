Edit Distance

Given two words word1 and word2, find the minimum number of operations required to convert word1 to word2.
You have the following 3 operations permitted on a word:
- Insert a character
- Delete a character
- Replace a character
Example 1:
Input: word1 = "horse", word2 = "ros"
Output: 3
Explanation:
horse -> rorse (replace 'h' with 'r')
rorse -> rose (remove 'r')
rose -> ros (remove 'e')
Example 2:
Input: word1 = "intention", word2 = "execution"
Output: 5
Explanation:
intention -> inention (remove 't')
inention -> enention (replace 'i' with 'e')
enention -> exention (replace 'n' with 'x')
exention -> exection (replace 'n' with 'c')
exection -> execution (insert 'u')

wiki

在计算语言学和计算机科学中，编辑距离是一种通过计算将一个字符串转换为另一个字符串所需的最小操作数来量化两个字符串（例如单词）彼此之间有多么不同的方式。编辑距离可以在自然语言处理中找到应用程序，其中自动拼写校正可以通过从字典中选择与所讨论单词的距离较小的单词来确定拼写错误的单词的候选者更正。在生物信息学中，它可以用于量化DNA序列的相似性，可以将其视为字母A，C，G和T的字符串。编辑距离的不同定义使用不同的字符串操作集。Levenshtein距离操作是字符串中字符的删除，插入或替换。作为最常用的度量标准，术语Levenshtein距离经常与编辑距离互换使用。

Types of edit distance

不同类型的编辑距离允许不同的字符串操作集。例如：

  1. Levenshtein距离允许删除，插入和替换;
  2. 最长公共子序列（LCS）距离仅允许插入和删除，而不允许替换;
  3. 汉明距离仅允许替换，因此，它仅适用于相同长度的字符串;
  4. Damerau–Levenshtein距离允许两个相邻字符的插入，删除，替换和移位;
  5. Jaro距离仅允许换位。

Definition

给定字母Σ上的两个字符串a和b（例如ASCII字符集，字节集[0..255]等），编辑距离d(a, b)是最小权重的编辑序列将a转换为b的运算。Levenshtein在1966年定义的最简单的一组编辑操作之一是：

  1. 插入单个符号。如果a = uv，则插入符号x会生成uxv。这也可以表示为ε→x，使用ε表示空字符串;
  2. 删除单个符号，会将uxv更改为uv(x→ε);
  3. 用单个符号x替换符号y≠x将uxv更改为uyv(x→y)。

Example

"kitten"和"sitting"之间的Levenshtein距离为3。将前者转换为后者的最小编辑脚本为：

  1. kitten → sitten (substitute "s" for "k")
  2. sitten → sittin (substitute "i" for "e")
  3. sittin → sitting (insert "g" at the end)

LCS距离（仅插入和删除）提供了不同的距离和最小的编辑脚本：

  1. kitten → itten (delete "k" at 0)
  2. itten → sitten (insert "s" at 0)
  3. sitten → sittn (delete "e" at 4)
  4. sittn → sittin (insert "i" at 4)
  5. sittin → sitting (insert "g" at 6)

总成本/距离为5次操作。

Solution

这道题的题意很清楚：求从单词word1转为单词word2所需要的最少变换次数。我看到这道题也有很多思绪，但是最后我还是没能做出来（思绪太乱了，最后也没理清楚。。后面我去讨论区看了大家的讨论，这题的最佳解决方案还是动态规划（我虽然思绪万千，但是没往动态规划想(⊙o⊙)…。然后我看到一个点赞比较多的讨论（[@anderson5's discussion](https://leetcode.com/problems/edit-distance/discuss/159295/Python-solutions-and-intuition)），我觉得写得挺好，这里引用一下他的解题思路。因为我觉得他的解题思路不仅仅是针对这道题，以后设计到同类型的题都会有启发，所以这里马一下：

anderson5’s thought

对于那些难以破解动态编程解决方案的人，我发现最简单的解决方法是首先从一个朴素但可行的递归实现开始。这样做是必不可少的，因为动态编程基本上是使用缓存进行递归的。通过这样的过程，对我们普通人来说，解答动态编程问题将变得更加容易。:)

思考过程

在给定两个字符串的情况下，我们的任务是找到为获得等效字符串而需要进行的最少转换数量。从一开始，似乎没有办法解决所有可能的问题，在这种情况下，可能性是指插入，删除或替换字符。递归通常是尝试所有可能性的不错选择。

每当我们编写递归函数时，我们都需要某种方法来终止，否则我们将通过无限递归最终导致堆栈溢出。对于字符串，要跟踪的是索引。我们需要两个索引，一个用于word1，一个用于word2。现在，我们只需要处理基本案例和递归案例。当我们遍历完两个字后会怎样？有些想法会告诉您，转换的最小次数只是插入其他单词的其余部分。这是我们的基本情况。当我们不使用任何一个字符串怎么办？我们要么匹配两个字符串中当前索引的字符，要么不匹配。在第一种情况下，我们不会产生任何损失，并且可以通过重复两个字符串的其余部分来继续比较其余字符串。在不匹配的情况下，我们可以插入，删除或替换。回顾一下：

  1. 基本情况：word1 =“”或word2 =“” =>返回其他字符串的长度；
  2. 递归的情况：word1 [0] == word2 [0] =>对word1 [1：]和word2 [1：]进行递归；
  3. 递归的情况：word1 [0]！= word2 [0] =>通过插入，删除或替换来递归。

在Python中：

class Solution:
    def minDistance(self, word1, word2):
        """Naive recursive solution"""
        if not word1 and not word2:
            return 0
        if not word1:
            return len(word2)
        if not word2:
            return len(word1)
        if word1[0] == word2[0]:
            return self.minDistance(word1[1:], word2[1:])
        insert = 1 + self.minDistance(word1, word2[1:])
        delete = 1 + self.minDistance(word1[1:], word2)
        replace = 1 + self.minDistance(word1[1:], word2[1:])
        return min(insert, replace, delete)

有了一个可用的解决方案，我们欣喜若狂，我们去提交我们的代码。一切都很好，直到我们看到可怕的红色文本为止……TIME LIMIT EXCEEDED。我们做错了什么？让我们看一个简单的示例，为了简洁起见，我将minDistance函数注释为md。

我们解决此问题的方法是通过缓存。我们将中间计算结果保存在一个字典中，如果我们再次处理相同的子问题，而不是再次执行相同的工作，我们将返回保存的值。这是记忆化的解决方案，我们从较大的子问题构建为较小的子问题（自上而下）。

class Solution:
    def minDistance(self, word1, word2, i, j, memo):
        """Memoized solution"""
        if i == len(word1) and j == len(word2):
            return 0
        if i == len(word1):
            return len(word2) - j
        if j == len(word2):
            return len(word1) - i

        if (i, j) not in memo:
            if word1[i] == word2[j]:
                ans = self.minDistance2(word1, word2, i + 1, j + 1, memo)
            else: 
                insert = 1 + self.minDistance2(word1, word2, i, j + 1, memo)
                delete = 1 + self.minDistance2(word1, word2, i + 1, j, memo)
                replace = 1 + self.minDistance2(word1, word2, i + 1, j + 1, memo)
                ans = min(insert, delete, replace)
            memo[(i, j)] = ans
        return memo[(i, j)]

当然，交互式实现通常比其递归实现要好，因为在递归调用的数量非常多的情况下，我们不承担炸毁堆栈的风险。我们还可以使用2D数组执行与缓存值的字典基本相同的操作。当我们这样做时，我们建立了从较小的子问题到较大的子问题（自下而上）的解决方案。在这种情况下，由于不再需要传统意义上的“重复发生”，因此我们使用基本约束来初始化2D表。该表的第一行和列具有已知值，因为如果一个字符串为空，我们只需添加非空字符串的长度，因为这是达到等效字符串所需的最少编辑次数。对于记忆和动态编程解决方案，时间复杂度均为O(mn)，空间复杂度为O(mn)，其中m和n分别是word1和word2的长度。

class Solution:
    def minDistance(self, word1, word2):
        """Dynamic programming solution"""
        m = len(word1)
        n = len(word2)
        table = [[0] * (n + 1) for _ in range(m + 1)]

        for i in range(m + 1):
            table[i][0] = i
        for j in range(n + 1):
            table[0][j] = j

        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i - 1] == word2[j - 1]:
                    table[i][j] = table[i - 1][j - 1]
                else:
                    table[i][j] = 1 + min(table[i - 1][j], table[i][j - 1], table[i - 1][j - 1])
        return table[-1][-1]

Linux | Git Error | gnutls_handshake() failed - The TLS connection was non-properly terminated

Kayleen ¬∑ 2020-6-18 ¬∑ Ê¨°ÈòÖËØª

问题

前天在Ubuntu18.04下用git克隆代码的时候出现了以下错误：

第一个错误

Failed to connect to 127.0.0.1 port 8888 。这个错误很明显，就是无法连接网络，这是因为我配置 vpn ，把 git 的代理改成了全局代理造成的。

解决方案

使用命令：git config --global --unset http.proxy 取消代理。

第二个错误

再次使用 git clone 克隆代码，发现出线了新的错误，但是我不确定是否还是因为我配置 vpn 造成的。后面谷歌了一下，有人说是 Ubuntu18.4 以上版本，由于 Ubuntu 对 git 默认使用的 libcurl4-gnutls-dev 库做了修改，导致 git 无法使用 SSL 进行连接。（但是我Ubuntu的版本是18.4，且之前可以用 git 正常克隆代码和管理。我猜测可能还是因为我配置vpn造成的，因为那天我就干了这个事。。

解决方案

重新编译并安装使用libcurl4–openssl–dev的git.

安装所需要的工具包

sudo apt-get update
sudo apt-get install build-essential fakeroot dpkg-dev
sudo apt-get install libcurl4-openssl-dev

安装日志部分截图：

配置 git 编译环境

因为我们要重新编译和安装 git，所以我们需要通过以下命令来获取来配置编译 git 的环境：

Note: build-dep causes apt-get to install/remove packages in an attempt to satisfy the build dependencies for a source package.

sudo apt-get build-dep git
sudo apt-get source git

如果出现以上错误，解决方案：打开Ubuntu中的 软件和更新 -> Ubuntu软件-> 把源代码勾选上。

更新 debian/control 和 debian/rules 文件

把debian/control中所有的libcurl4-gnutls-dev改为libcurl4-openssl-dev，并删除debian/rules中的TEST = test。

cd git-2.17.1/
sudo sed -i 's/libcurl4-gnutls-dev/libcurl4-openssl-dev/g' debian/control
sed -i 's/TEST =test//g' debian/rules    # 删除 debian/rules 文件中 TEST = test

编译和构建安装包

sudo dpkg-buildpackage -rfakeroot -b

重新安装编译好的git

sudo dpkg -i git_2.17.1-1ubuntu0.7_amd64.deb   # 根据自己的版本更改。

OK搞定！

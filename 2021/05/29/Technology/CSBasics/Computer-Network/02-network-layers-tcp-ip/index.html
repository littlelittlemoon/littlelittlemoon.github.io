写到前面的话

稳扎稳打，回顾基础！

上一篇文章我们回顾了 OSI 七层协议（参考模型），本篇文章主要在上一篇文章的基础上，介绍 TCP/IP 的 4、5 层模型。OSI 参考模型是网络的标准层次划分，TCP/IP 模型不过是在该基础上，在具体实现上对分层做了一点点调整。实际上 TCP/IP 中的各种协议也能对应到OSI参考模型当中。下面我们就带着上篇文章的记忆，开始学习 TCP/IP 模型吧！

TCP/IP 与 OSI 参考模型

从上图中我们不难看出，TCP/IP 与 OSI 在分层模块上稍有区别。OSI参考模型注重 “通信协议必要的功能是什么”，而 TCP/IP 则更强调 “在计算机上实现协议应该开发哪种程序”

相同点

层次划分思想相同

两种体系结构都是以协议栈（不同协议）形成的层次结构，协议栈中的协议彼此独立，以简化各种网络协议的设计，只需要为不同协议提供关联接口即可。

层次结构相似

这两个体系结构虽然总的层数和对应层次名称都有所不同，但总体层次结构极其相似。TCP/IP 协议体系结构中的网络接口层对应了 OSI 的数据链路层和物理层。TCP/IP 协议体系结构的应用层对应 OSI 中的应用层、表示层和会话层。TCP/IP 中的网络层和传输层与 OSI 中的网络层和传输层一样。在这两种结构中，传输层以下都属于通信子网部分，传输层及以上各层都提供了端到端、与网络无关的服务，属于资源子网部分。

核心组成一样

两种体系结构中都定义了服务、接口和协议三个重要核心概念：

  1. 服务：定义了各层应该做些什么，要提供哪些功能。
  2. 接口：为对应的上层提供了获取本层服务的逻辑接口，规定了有哪些参数可以使用，以及使用这些参数的结果是什么。
  3. 协议：它是各层服务功能的具体实现者。当然同一服务在不同网络中可以用不同的协议来实现。各层中的协议各自实现自己的功能，并不影响其他层，任何的一层都只为相邻的上一层提供服务。

区别

适用范围不同

OSI 是先有理论模型，后面才开发标准化协议，所以 OSI 不偏重于任何特定的网络类型，具有最广泛的理论参考性，是一个理想化的模型。而 TCP/IP 协议体系结构则相反，它是在 TCP/IP 协议族的基础上，专门针对这些协议进行的功能描述和层次划分，与协议的关系非常紧密，仅适用于 TCP/IP 网络，最具实践性。

层次结构不同

TCP/IP 协议体系结构中没有会话层和表示层，它们的功能合并在应用层之中。另外，OSI 中的物理层和数据链路层的功能在 TCP/IP 协议体系结构中合并到网络接口层，尽管实际上在 TCP/IP 协议体系结构中对这个层中的具体功能并没有明确规定，实际上这层功能就是 OSI 物理层和数据链路层的功能。

网络通信模式不同

OSI 的网络层同时支持无连接和面向连接的网络通信，TCP/IP 模型的网络层只提供简单灵活的、无连接、尽最大努力交付的数据报服务。

通信协议不同

OSI 是一种开放型的适用于所有类型计算机网络的理想化体系结构模型，所以它的通信协议不仅非常多而且类型复杂，适用于各类网络。但是，由于现在网络系统设计者通常不是参考 OSI，而是以目前占据了绝大部分市场的 TCP/IP体系结构作为设计参考，所以现在 OSI 中的许多通信协议都已过时。

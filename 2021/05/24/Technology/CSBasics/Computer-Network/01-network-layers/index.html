协议

协议 (protocol) 定义了在两个或多个通信实体之间交换的报文的格式和顺序，以及报文发送和/或接收一条报文或其他事件所釆取的动作。掌握计算机网络领域知识的过程就是理解网络协议的构成、原理和工作方式的过程。

协议分层

协议分层就如同计算机软件中的模块化开发。通过分层能够细分通信功能，更易于单独实现每个分层的协议，并界定各个分层的具体责任和义务。

OSI 七层模型（参考模型）

ISO 在制定标准化 OSI 之前，对网络体系结构进行了充分的讨论，最终提出作为通信协议设计指标的OSI 参考模型。在这一模型中，每个分层都接收由它下一层所提供的特定服务，并负责为自己的上一层提供特定服务。上下层之间进行交互时所遵循的约定叫做 “接口”。同一层之间的交互所遵循的约定叫做 “协议”。如下图所示：

应用层（7）

功能

针对特定应用的协议。为应用程序提供服务并规定应用程序中通信相关的细节。

主要协议
  1. FTP（文件传送协议）
  2. Telnet（远程登录协议）
  3. DNS（域名解析协议）
  4. SMTP（邮件传送协议）
  5. POP3协议（邮局协议）
  6. HTTP协议（Hyper Text Transfer Protocol）
举例

用户 A 在主机 A 上新建一封电子邮件，指定收件人为 B，并输入邮件内容为 “早上好”。

从用户输完所要发送的内容并点击 “发送” 按钮的那一刻开始，就进入了应用层协议的处理。该协议会在所要传送数据的前端附加一个首部（标签）信息标明：

- 邮件内容：“早上好”
- 收件人：“B”

主机 B 上的应用收到由主机 A 发送过来的数据后，分析其数据首部和数据正文，并将邮件保存到硬盘或是其他非易失性存储器（数据不会因断电而丢失的存储设备）设备进行相应处理。如果主机 B 上收件人因为各种原因无法接受新的邮件，则返回错误给发送方。这类异常的处理也属于应用层需要解决的问题。

由此可见，主机A与主机B通过各自应用层之间的通信，最终实现邮件的存储。

表示层（6）

功能

设备固有数据格式和网络标准数据格式的转换。将应用处理的信息转换为适合网络传输的格式，或将来自下一层的数据转换为上层能够处理的格式。主要负责数据格式的转换。

表示层的数据转换包括数据的加密、压缩、格式转换等。

举例

表示层更关注数据的具体表示形式。如果用户A与用户B所使用的的邮件客户端软件完全一致，就能够顺利收取和阅读邮件。但现实生活中这种情况不大可能，现在出了个人电脑，还有其他设备如只能手机也都能连接到网络。如何让它们之间能够相互读取通信数据变得非常重要。解决这类问题的方法是：
- 首先发送端利用表示层，将数据转换为 “网络通用的标准数据格式” 之后再发出去； - 接收端收到数据以后将这些网络标准格式的数据恢复为 “该计算机特定的数据格式” ，然后再进行处理。

对于刚刚的例子中的 “早上好” 这段文字根据其编码格式被转换成为了 “统一网络数据格式”。表示层与表示层之间为了识别编码格式也会附加首部信息，从而将实际传输的数据转交给下一层去处理。

会话层（5）

功能

通信管理。负责建立和断开通信连接（数据流动的逻辑通路）(管理主机之间的会话进程，即负责建立、管理、终止进程之间的会话)，以及数据的分割等数据传输相关的管理，利用在数据中插入校验点来实现数据的同步。管理传输层以下的分层。

举例

假设用户 A 新建了 5 封邮件准备发送给用户 B，这 5 封邮件的发送顺序可以有很多种： - 每发一封邮件建立一次连接（通信连接）； - 建立一次连接将 5 封邮件连续发送出去； - 同时建立好 5 个连接，将 5 封邮件同时发给对方。

决定采用何种连接方式是会话层的主要责任。会话层也像应用层和表示层那样样，在其收到的数据前端附加首部或标签信息后在转发给下一层。这些首部信息或标签信息记录着数据传送顺序的信息。

传输层（4）

功能

管理两个节点之间的数据传输，无需在路由器上处理。负责可靠传输（确保数据被可靠地传送到目标地址）。

主要协议
  1. TCP 协议（Transmission Control Protocol，传输控制协议）
  2. UDP 协议（User Datagram Protocol，用户数据报协议）

重要设备：网关

举例

会话层只对何时建立连接、何时发送数据等问题进行管理，并不具备实际传输数据的功能。传输层进行实际的建立和断开处理。传输层为确保所发出的数据到达目标地址，会在通信两端的计算机之间进行确认，如果数据没有到达，它会负责进行重发。

例如，主机 A 将 “早上好” 这一数据发送给主机 B。期间可能会因为某些原因导致数据被破坏，或由于发生某种网络异常致使只有一部分数据到达目标地址。假设主机 B 只收到了 “早上” 这一部分数据，那么它会在收到数据后将自己没收到 “早上”之 后的那部分数据的事实告诉主机 A。主机得知这个情况后就会将后面的 “好” 重发给主机 B ，并再次确认对端是都收到。

此外，传输层还要处理端到端的差错控制和流量控制问题。传输层会根据通信子网的特性，最佳的利用网络资源，为两个端系统的会话层之间，提供建立、维护和取消传输连接的功能，负责端到端的可靠数据传输。在这一层，信息传送的协议数据单元称为段或报文。网络层只是根据网络地址将源结点发出的数据包传送到目的结点，而传输层则负责将数据可靠地传送到相应的端口。

网络层（3）

功能

地址管理与路由选择。负责将数据传输到目标地址。目标地址可以是多个网络通过路由器连接而成的某一个地址。因此这一层主要负责寻址和路由器选择。

主要协议
  1. IP 协议（Internet Protocol，因特网互联协议）
  2. CMP 协议（Internet Control Message Protocol，因特网控制报文协议）
  3. ARP 协议（Address Resolution Protocol，地址解析协议）
  4. RARP 协议（Reverse Address Resolution Protocol，逆地址解析协议）

重要设备：路由器。

举例

网络层的作用是在网络与网络相互连接的环境中，将数据从发送端主机发送到接收端主机。

如上图所示，两端主机之间有总多数据链路，能够将数据从主机 A 发送到主机 B 都是网络层的功劳。在实际发送数据时，目的地址至关重要。这个地址是进行通信的网络中唯一指定的序号。基于这个地址，就可以在网络层进行数据包的发送处理。而有了地址和网络层的包发送处理，就可以将数据发送到世界上任何一台互联设备。网络层也会将其从上层收到的数据和地址信息等一起发送给下面的数据链路层，进行后面的处理。

数据链路层（2）

功能：互连设备之间传送和识别数据帧。1）负责物理层面上互连的、节点之间的通信传输。例如与 1 个以太网相连的 2 个节点之间的通信。2）将 0、1 序列划分为具有意义的数据帧传送给对端（数据帧的生成与接收）

通信传输实际上是通过物理的传输介质实现的。数据链路层的作用就是在这些通过传输介质互连的设备之间进行数据处理，基本数据单位为帧。

主要协议
  1. 以太网协议

重要设备：1）网桥； 2）交换机

物理层（1）

功能：以 0、1 代表电压的高低、灯光的闪灭，负责 0、1 比特流与典雅的高低、光的闪灭之间的互换（激活、维持、关闭通信端点之间的机械特性、电气特性、功能特性以及过程特性）。界定连接器和网线的规格。该层为上层协议提供了一个传输数据的可靠的物理媒体。简单的说，物理层确保原始的数据可在各种物理媒体上传输

MAC 地址

物理层中，将数据的 0、1 转换为电压和脉冲光传输给物理的传输介质，而相互直连的设备之间使用地址实现传输。这种地址被称为 MAC(Media Access Control, 介质访问控制) 地址。也可称为物理地址或硬件地址。采用 MAC 地址，是为了识别连接到同一个传输介质上的设备。因此在这一分层中将包含 MAC 地址信息的首部附加到从网络层转发过来的数据上，将其发送到网络。

重要设备：1）中继器（Repeater，也叫放大器）； 2）集线器

举例

接收端主机 B 上的处理流程正好与主机 A 相反，它从物理层开始将接收到的数据组成发给上一分层进行处理，从而使用户 B 最终在主机 B 上使用邮件客户端软件接受用户 A 发送过来的邮件，并可以读取相应内容为 “早上好”。
